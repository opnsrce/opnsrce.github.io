The ability to solve new problems efficiently is the hallmark of a good developer. When interviewing dev candidates, I don&#8217;t necessarily look for their experience with a particular framework or API. Most of the time, I&#8217;m looking at how they go about approaching a problem they&#8217;ve (most likely) never seen before. You&#8217;ve probably encountered questions like &#8220;write a function that finds x number in the <a class="zem_slink" href="http://en.wikipedia.org/wiki/Fibonacci_number" title="Fibonacci number" rel="wikipedia" target="_blank">Fibonacci sequence</a>&#8221; or the ubiquitous &#8220;<a class="zem_slink" href="http://en.wikipedia.org/wiki/Fizz_buzz" title="Fizz buzz" rel="wikipedia" target="_blank">FizzBuzz</a>&#8221; puzzle. These questions and others like it are designed to give the interviewer insight into how you think about a problem and the kinds of solutions you&#8217;ll come up with.
  
<!--more-->

As a developer, the frustrating part about being judged on my problem solving skills is that I find such skills difficult to improve. Problem solving is a skill that improves by, well, _solving problems_. While books can certainly be helpful in making you a better problem solver (<a href = "http://www.levihackwith.com/recommended-reading-how-to-think-like-a-programmer/">here&#8217;s one I&#8217;ve recommended</a>), what you really need to do is to find a resource that allows you to hammer away at as many programming problems as you can handle: <a href = "http://www.codewars.com/">Code Wars</a>. Code Wars allows you to progress through a series of increasingly difficult programming puzzles in whatever language you&#8217;d like. I highly recommend it for devs looking to improve their problem solving skills.
